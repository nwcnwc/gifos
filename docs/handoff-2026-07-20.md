State of the meeting-mesh work after the join/transport session. Written to be

## What is live on gifos.app right now

All four of these are deployed and smoke-tested (4 bots converge 4/4 on
production):

1. **The relay fallback is gone.** `mesh-wire.js` no longer sends arbitrary mesh
   frames over the relay when a peer has no DataChannel. It lied about E2
   first-hand liveness, hid the link bug below, and parked ~0.5 frames/s of
   heartbeat per unbuilt link onto one socket.
2. **Every neighbour the mesh names is dialled.** `renderFromOcc` used to dial
   only peers that passed `alive()` (heard-from / connected / first-hand live),
   making contact a precondition for the dial that establishes contact. Row
   mates escaped it via the admission handshake; **column mates never could**,
   which is why rows worked and columns did not.
3. **A named neighbour is never "absent".** Regression from (2): the freshly
   dialled peer was marked absent on the same pass, the sweeper dropped it, and
   every re-offer path is gated on `!absentSince`.
4. **One door for the relay.** `RELAY_OK` (a frame-type allowlist) is deleted.
   Five named functions in `mesh-wire.js` are the only things that may touch the
   socket, each enforcing its own precondition — `KNOCK_FOR_THE_GREETER_LIST`,
   `REGISTER_MYSELF_AS_A_GREETER`, `AT_THE_DOOR_ASKING_TO_BE_LET_IN`,
   `ANSWERING_SOMEONE_AT_THE_DOOR`, `RELAY_FIRST_CONTACT_SIGNALING`.

`relay/` (the Cloudflare Worker) is **byte-identical to what is deployed** —
nothing is pending a wrangler deploy. Its caps stand by decision:
`MAX_SOCKETS_PER_IP = 8` in production, off in test via `RELAY_DEV=1`.

## Operating rules learned the hard way

- **`site/` AUTO-DEPLOYS on push.** Run `test/batteries/join.sh` BEFORE pushing
  `site/js/mesh-wire.js`, `site/js/mesh.js` or `site/meet.html`. This was
  violated twice in one session; both times it shipped a regression.
- **Test on nvidia-laptop** (8 cores). Penguin is weak — above ~9 bots it
  invents failures that look exactly like mesh bugs, and it also hosts the
  tooling. Three separate "bugs" this session were the test fleet:
  a stripped Chromium, a swarm arg parser that silently retargeted runs at
  production, and box exhaustion.
- **Real Chrome only**: `/opt/google/chrome/chrome`. A stripped playwright build
  loads the page and dies on the media path — and seats fine under `--lite`,
  so a lite run "proving the fleet is healthy" proves nothing about a video run.
- **`RELAY_DEV=1`** for any local relay: lifts the per-IP, join-rate, session
  and frame caps.
- **Never combine `pkill -f <pattern>` with a command containing that pattern** —
  it kills its own shell. Put the launch in a separate call or a script file.
- **Know what the sim can and cannot see.** It DOES inject real faults — loss,
  latency, link severance, mutually unreachable subnets, and a targeted
  `sever A B T` that both ends observe — so hostile networks belong in
  `sim/repro-adversary.sh`, at hundreds of seats and deterministically. What it
  does not model is link ESTABLISHMENT: a pair that could talk but has no
  DataChannel open yet and needs signaling to get one. Every expensive bug this
  month lived in exactly that gap, which is why browser runs must assert link
  completeness rather than seat counts.
