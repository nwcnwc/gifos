## Loss wedge design — LOCKED (Nathan 2026-07-21)

**Problem (plain):** under ~10% packet loss, a “go sit at X” can be lost after the room already treats X as taken → permanent ghost → room stuck at ~5 seats. Repro:

```bash
printf "net loss=0.10\njoinmode burst\ninit 60\nconverge 400000\nstate\nbad\nseat 0\nseat 6\nquit\n" | /tmp/mesh --service
# seat 0: /1.0=6 phantom; seat 6: searching forever
```

**Direction: three-state seat notes** (empty / sitting-down / seated).

| State | Meaning | Who sets it |
|---|---|---|
| empty | free | default / after clear |
| **sitting-down** | assigned, not confirmed | whoever writes the PLACE-equivalent note (the seat-assigner — same role as admitter for that cell, **not** the greeter/door) |
| **seated** | really there | **the joiner themselves** when they take the seat |

**Rules Nathan locked:**

1. **Self-confirm:** only the person sitting upgrades sitting-down → seated (their take / HELLO / equivalent). Not “assume after N ticks” alone.
2. **Mass join must stay fast:** hand-offs and filling the **rest of the same row** are **not** blocked on the head being fully seated — only on the soft “sitting-down” note existing is fine. **Do not** reintroduce a firstHandLive / “must be seated” gate on every hand-off (that made healthy 1000-person bursts unusable).
3. **Who rechecks:** the party that wrote the **sitting-down** mark (the assigner for that cell). Not “greeter” unless they also assigned. Soft TTL backstop OK so marks cannot live forever if recheck is skipped.
4. **Row-then-depth gate (mass entry):**  
   - **Allowed before head is seated:** fill the **entire row** while the head is still only sitting-down (or seating in parallel).  
   - **Not allowed until head is seated:** spill **past that row** (children / next row / deeper frontier). That is the gate: “don’t hand the row head would-be children until head is **seated**.”  
   - Recheck of sitting-down happens on a short timer **before** that deeper hand-off is allowed.
5. **Double claim:** non-adversarial nodes still yield by the existing simple rule (lower id / tenure as already in E2). Soft marks may both expire if neither confirms.

**Explicitly rejected:** pure firstHandLive admitter hand-off gate; PLACE TTL expire alone (scan never reached the cell).

**Implement when A is opened:** sim-first (`noteS1` / admit / PLACE / serveFind paths), pin 10% loss repro + healthy burst N=100/1000 convergence not regressing, then port `mesh.js`. Keep join battery green.
