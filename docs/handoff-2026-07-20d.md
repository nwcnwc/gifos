### Important semantics correction on `founded`

The relay's `founded` flag is **not** "this session already has a meeting".
From `relay/src/relay.js` knock handler:

> `founded` — **this knocker** met an empty registry and **minted** the
> genesis instance (R3).

So:

| greeters reply | meaning |
|---|---|
| empty + `founded:true` | *you* just minted genesis → wire delivers → seat takes `0/0.0` |
| empty + `founded:false` | mint gap / no admit (genesis may already exist with empty pool) → hold |
| non-empty + `founded:false` | normal join path (genesis already set by someone else) |

That is why late joiners always show `founded:false` with full lists — they
are joining an existing genesis, not founding. R3/R6 fragment mint needs
empty + **you** founded, which only happens when the relay has **no**
genesis hash yet. Greeter-pool expiry alone cannot produce that
(`test/mesh/greeter-expiry.js` already proved empty-pool + existing genesis
⇒ hold, not mint). Session 2's "healthy late joiner founds `0/0.0`" is
therefore either rare race on a clean session, a different sid, or was
entangled with the late-join wiring deadlock (now fixed). Instrumentation
stays so the next flake prints the smoking gun.
