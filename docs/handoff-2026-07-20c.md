### Product root causes (stacked)

After the relay chokepoint (session 1: no mesh-over-relay fallback, dial every
named neighbour), several independent holes stacked into "have-local-offer
forever / pairs=[]":

| hole | what it did | fix |
|---|---|---|
| `forceSeat` pre-seeded `occ`, then `doMove` **cleared** it | ARRANGED joiner landed with **zero** `linkPeers` (`pairs=[]`) when it had already seated naturally | pass seed as **`nbrs` into `take`/`doMove`** (same as PLACE) |
| `meshSendDC` returned `false` for targets still in `relaySocketed` | expected sealed-relay fallback that **deliver() no longer has** — HELLO/PHONE became silence | always **sponsor** when no open DC |
| hop `fwdDedup` stamped on **entry** | multi-door fan used one envelope id; a dead-end greeter "saw" it and blocked a later good path | stamp only after **successful handoff** |
| `nextHopDc` accepted zombie pairs | local `connected`/`dc open` while the far side had dropped the peer — `dcSend` no-op | require **live** connected+open DC; mutual-friend / last-resort alt |
| `wired()` true after **any** one DC | deep seat dropped relay while other named neighbours still unbuilt | keep/reopen socket until **every** `linkPeer` is wired |
| lower-id never offered | id-order assumes both sides know each other; if HELLO never lands, higher id never dials | after grace, **either** side may offer; glare = polite rollback |
| door always took `doors[0]` (geometric parent) | parent often held a zombie edge | prefer peers that **claim** the target in `connsOf`; fsig fans 3 doors; HELLO/CLAIM fan **all** greeter doors |

Also added DEBUG `learnOcc(coord, pid)` — injects one occ entry (the effect of a
delivered HELLO). The latejoin ARRANGED leg uses it so the socketless side
knows the joiner the way PLACE would; the thing under test remains offer/answer
over §FWD.

### Drill / fleet fixes (so the bug is measurable)

`test/drills/e2e-latejoin.js` was itself a trap (session 2 called the ARRANGED
leg "real" once it arranged, then the product still failed; later runs failed
to arrange at all):

1. **`RELAY_DEV=1`** on the spawned relay — without it the production frame
   meter (3 frames/s) silently drops door/ICE traffic and looks exactly like
   the deadlock. `adversary-room.js` already had this; latejoin only had
   `TRUSTED_IPS` (per-IP sockets, **not** the frame meter).
2. **`occSeed` from direct page snaps**, not `probeTree` — mesh gossip often
   returned `census=1` once deep seats were socketless.
3. **ARRANGED does not require natural seating** — mesh node ready → `forceSeat`
   (take or doMove+nbrs) → `learnOcc` on each socketless target.
4. Opportunistic LATE1 no longer fails the suite when seating gave it no
   socketless neighbour (ARRANGED is the gate).
5. Prefer **`MEET_CHROME=/opt/google/chrome/chrome`** and run on **nvidia**.

### 3. Loss wedge — diagnosed, needs Nathan's call

Unchanged from session 2. Repro:

```bash
printf "net loss=0.10\njoinmode burst\ninit 60\nconverge 400000\nstate\nbad\nseat 0\nseat 6\nquit\n" | /tmp/mesh --service
# seat 0: /1.0=6 ; seat 6: state=searching forever
```

resurrection scans an empty row below → every FIND is handed to a never-seated

`firstHandLive` (correct seating, unusable healthy convergence), (b) PLACE TTL
expire (scan never reaches the cell).
